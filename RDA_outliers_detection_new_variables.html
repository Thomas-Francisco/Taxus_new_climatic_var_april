Candidate selection: Redundancy analysis(RDA)

In this script, we will perform an RDA (not accounting for population structure): Redundancy analysis are multivariate canonical analysis that are halfway between PCA and multiple regression. Indeed, it performed an analysis similar to a multiple regression, the major differences is that RDA perform a multiple regression with a matrix of response variables and not just one. It’s a canonical analysis because this analysis will create canonical axis similar to a PCs. The explanation of this analysis is that we first performed linear regressions between the explanatory variables and the responses variables then we performed on the fitted values (result of the linear regressions) a PCA to create the canonical axis (RDA axis) to reduce the number of dimensions. These canonical axis are the results of the linear combinations of the explanatory variables. It’s like a PCA and a multiple regression because in the end, we have results similar to PCA but with fitted values enabling us to extrapolate the model (following a linear relationship). In the end, RDA is a multivariate analysis following linear hypothesis that cannot be corrected for population structure (see pRDA for correction) Sources: Legendre and Legendre 2012, Capblancq 2021, workshop redundancy analysis of the Quebec centre for Biodiversity Science (https://r.qcbs.ca/workshop10/book-en/redundancy-analysis.html)

3. Identification of outliers

The next step is to perform the genome scan procedure to calculate the Mahalanobis distance and then calculate qvalues and pvalues associated to this distance that will be used as threshold for candidates selection.

We used the rdadapt function from Capblancq et al 2021:

rdadapt<-function(rda,K)
{
  zscores<-rda$CCA$v[,1:as.numeric(K)]
  resscale <- apply(zscores, 2, scale)
  resmaha <- covRob(resscale, distance = TRUE, na.action= na.omit, estim="pairwiseGK")$dist
  lambda <- median(resmaha)/qchisq(0.5,df=K)
  reschi2test <- pchisq(resmaha/lambda,K,lower.tail=FALSE)
  qval <- qvalue(reschi2test)
  q.values_rdadapt<-qval$qvalues
  return(data.frame(p.values=reschi2test, q.values=q.values_rdadapt))
}

We also perform the outlier detection based on the loadings of loci along the RDa axis to search for extreme loadings on each retained axis following Forester et al. (2018).

load.rda <- scores(RDA_env, choices=c(1:3), display="species")

hist(load.rda[,1], main="Loadings on RDA1")

hist(load.rda[,2], main="Loadings on RDA2")

Function from Forester et al. (2018).

#function to do it
outliers_function <- function(x,z){
  lims <- mean(x) + c(-1, 1) * z * sd(x)     # find loadings +/-z sd from mean loading     
  x[x < lims[1] | x > lims[2]]               # locus names in these tails
}

cand1 <- data.frame(outliers_function(load.rda[,1],3)) # 3
colnames(cand1) <- c("pvalues"); cand1$snp_names <- row.names(cand1)
cand2 <- data.frame(outliers_function(load.rda[,2],3)) # 51
colnames(cand2) <- c("pvalues"); cand2$snp_names <- row.names(cand2)

#names of the outliers
outliers_extremness <- rbind (cand1,cand2)

nrow(outliers_extremness)
## [1] 144
