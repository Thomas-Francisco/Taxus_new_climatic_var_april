Correcting for population structure

pRDA is a version of the Redundancy analysis used to correct for some variables while performing an RDA. The steps are exactly the same as an pRDA but 1 step is performed before. Indeed, the goal of the pRDA is to perform an RDA while correcting for some other variables that could have an impact on the responses or/and the explanatory variables.To account for that, pRDA first perform a linear regression between the response var and the explanatory var and with the conditioned variables (extra variables that could impact the response/ explanatory variables) as covariates. Then we performed the same step as an RDA but on the residual of the response variables and the explanatory variables because the residuals are the variance not explained by the conditioned variables -> this enable the model to remove the effects of the conditioned variables on the responses and explanatory variables. Sources: Legendre and Legendre 2012, Capblancq 2021, workshop redundancy analysis of the Quebec centre for Biodiversity Science (https://r.qcbs.ca/workshop10/book-en/redundancy-analysis.html)

We will correct for population structure using only the 2 firsts PCa axis to avoid over correcting and to be consistent with other analysis (LFMM)

3. Identification of outliers

The next step is to perform the genome scan procedure to calculate the Mahalanobis distance and then qvalues and pvalues in the same way as in the RDA script.

rdadapt<-function(rda,K)
{
  zscores<-rda$CCA$v[,1:as.numeric(K)]
  resscale <- apply(zscores, 2, scale)
  resmaha <- covRob(resscale, distance = TRUE, na.action= na.omit, estim="pairwiseGK")$dist
  lambda <- median(resmaha)/qchisq(0.5,df=K)
  reschi2test <- pchisq(resmaha/lambda,K,lower.tail=FALSE)
  qval <- qvalue(reschi2test)
  q.values_rdadapt<-qval$qvalues
  return(data.frame(p.values=reschi2test, q.values=q.values_rdadapt))
}

We also perform the outlier detection based on the loadings of loci along the pRDA axis to search for extreme loadings on each retained axis following Forester et al. (2018).

load.rda <- scores(pRDA_env, choices=c(1:3), display="species")

hist(load.rda[,1], main="Loadings on RDA1")

hist(load.rda[,2], main="Loadings on RDA2")

#function to do it
outliers_function <- function(x,z){
  lims <- mean(x) + c(-1, 1) * z * sd(x)     # find loadings +/-z sd from mean loading     
  x[x < lims[1] | x > lims[2]]               # locus names in these tails
}

cand1 <- data.frame(outliers_function(load.rda[,1],3)) # 3
colnames(cand1) <- c("pvalues"); cand1$snp_names <- row.names(cand1)
cand2 <- data.frame(outliers_function(load.rda[,2],3)) # 51
colnames(cand2) <- c("pvalues"); cand2$snp_names <- row.names(cand2)

#names of the candidate
outliers_extremness <- rbind (cand1,cand2)
